# Contributing

## Git workflow

All substantive work MUST follow `ai-specs/specs/git-workflow.mdc` (also summarized in `ai-specs/specs/base-standards.mdc`):

1. **Create a new feature branch** from the default branch for each feature, fix, or spec update batch.
2. **Open a Pull Request** when ready.
3. The **maintainer must review and approve** the PR before merge. Do not merge your own PR without explicit maintainer approval.

## Scope of this repository

Changes here should stay focused on **requirements**, **OpenSpec artifacts**, **documentation**, and **AI/coding standards**. Do not add backend or frontend application code to this repository; link to the code repos from PR descriptions instead.

## Language

Per `base-standards.mdc`, technical artifacts (commits, PR titles/descriptions, spec text in this repo) should be in **English**.
